Build Commands
⚡
!build #builds
Ad-hoc build trigger. Fires sasmaster-build.sh immediately. Retries once after 5 min on failure, escalates if it fails again.
Firessasmaster-build.sh via Claude Code non-interactively
On successPosts "Build complete — check DONE_LOG.md"
On failureSchedules retry in 5 min → escalates to Slack on second failure
Slack response

Claude Code is awake. Stand by...
✅ Build complete — [timestamp]
📋
!task HIGH | MED | EXPLORE <description> #builds
Inject a task into TASKS.md. Will execute on the next build cycle (cron or manual !build).
Priorities HIGH executes next build   MED queued   EXPLORE research-only
Appends to~/SaSMaster/TASKS.md with timestamp
Example!task HIGH wire TMDB tv_series to parent key resolver
Slack response

> wire TMDB tv_series to parent key resolver
Added to TASKS.md. Will execute on next build.

Infrastructure
🔌
Socket Mode WebSocket
Always-on connection to Slack. Free tier sufficient. No outbound port required — Slack pushes events to JARVIS. Reconnects automatically on disconnect.
🚀
launchd Auto-Start
Registered as com.sasmaster.jarvis. Starts automatically on Mac boot. Restart: launchctl stop/start com.sasmaster.jarvis
📂
Approval Queue File
JSON queue at ~/SaSMaster/pending-approvals.json. Loaded atomically on each !approve / !skip call. Persists across JARVIS restarts.
🔄
Build Retry Logic
On failure: schedules 1 retry after 5 min. If retry also fails: posts escalation alert to #builds. No infinite loops — max 2 attempts per trigger.
🔒
Channel Allowlist
Commands only processed from ALLOWED_CHANNELS (builds + intel + content). Messages from other channels silently ignored. Bot ID filtered to prevent self-loops.
👤
JARVIS Identity
Runs as U0AUXSGSSJU (JARVIS user, owns SLACK_BOT_TOKEN). This is NOT the @SaSMaster Bot app — different token, different user ID.
