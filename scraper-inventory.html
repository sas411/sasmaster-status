TMDB Scrapers (Direct)
1
tmdb-bulk
~/SaSMaster/scripts/load-tmdb-to-s3.js
SOURCEapi.themoviedb.org/3 + files.tmdb.org/p/exports/
SCHEDULECOMPLETE — 1.18M records landed Apr 24, 2026
OUTPUTtmdb_dev/movies/ · tmdb_dev/tv_series/ · tmdb_dev/people/
FORMATNDJSON raw (STANDARD_IA)
RATERATE_LIMIT_MS=25 → ~40 req/sec (TMDB cap ~50/sec)
LOCK~/SaSMaster/data/tmdb-load.lock — kill keeper PID first
CHECKPOINT~/SaSMaster/status/tmdb-progress.json
Known Failures
  • HTTP 429 over burst limit
  • Lockfile collision between keeper loop and cron
  • S3 auth expiry mid-run (batches lose write access)
  • Partial batch writes (always write tmp → mv)
aws s3 ls s3://sasmaster-2026/tmdb_dev/movies/ | wc -l cat ~/SaSMaster/status/tmdb-progress.json
2
tmdb-delta
~/SaSMaster/scripts/load-tmdb-deltas.js
SOURCEapi.themoviedb.org/3/{movie,tv,person}/changes (15-day window)
SCHEDULEBiweekly — 1st and 16th of month 0 0 1,16 * *
OUTPUTtmdb_dev/deltas/
FORMATNDJSON raw
RATERATE_MS=30 → ~33 req/sec; safety page cap=500
DEPENDStmdb-bulk must have completed
Known Failures
  • Missing records if run offset >24h from schedule
  • Page cap (500) hit during high-churn periods (award seasons)
  • HTTP 429 during peak TMDB activity
aws s3 ls s3://sasmaster-2026/tmdb_dev/deltas/ --recursive

tmdb-tv-seasons
~/SaSMaster/scripts/load-tv-seasons-to-s3.js
SOURCEapi.themoviedb.org/3/tv/{id}/season/{n}
append: external_ids,videos,images
SCHEDULEChained — runs after tmdb-bulk TV phase completes
OUTPUTtmdb_dev/tv_seasons/YYYY-MM-DD/batch_NNNNNN.ndjson
RATERATE_MS=30 → ~33 req/sec
DEPENDStmdb-bulk TV phase — tmdb_dev/tv_series/ populated
NOTE~300K series × avg seasons = millions of API calls. Idempotent via S3 batch existence check.
Known Failures
  • S3 auth expiry on long runs
  • Season 0 ("Specials") has irregular episode numbering
  • Series with >50 seasons = oversized batch files
cat ~/.checkpoints/tv-seasons.json aws s3 ls s3://sasmaster-2026/tmdb_dev/tv_seasons/ --recursive | wc -l
