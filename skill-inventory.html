Self-Learning Pipeline (SLC)
How Skills Evolve — 3-Tier Compounding Loop
Every session builds the next session. Tier promotions run automatically via Fri 5:50PM cron (skill-review.sh). Manual patches via Claude Code for immediate session-critical updates (like today's ASK portal lessons).
📝
T1 — Session Capture
DONE_LOG.md → FEEDBACK.md → learnings.md. Every surprising result, anti-pattern, or confirmed approach gets captured. Threshold: ≥1 occurrence or Shiv-confirmed.
capture-learnings.sh
📚
T2 — Curation Gate
learnings.md → LESSONS.md. Quality gate: is this a real pattern (not just a one-off)? Does it generalize across sessions? Would future-me need this in a cold start?
LESSONS.md quality gate
🧠
T3 — Skill Promotion
LESSONS.md → SKILL.md. Patterns seen ≥3× promoted to permanent skills. Friday 5:50PM cron. Urgent patches (like today's ASK session) applied immediately via Claude Code skill patch cards.
skill-review.sh (Fri 5:50PM)
