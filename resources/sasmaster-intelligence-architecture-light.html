MEMORY

SaSMaster operates across four interlocking memory systems — each serving a distinct timescale and purpose. Claude.ai Memories, CLAUDE_MEMORY.md, and TASKS.md are live. The Obsidian vault mirrors operational state for human-readable handover.

auto-generated · claude.ai
Persistent semantic memories derived from every conversation. Captures Shiv's working style, project context, sprint priorities, product vision, and long-term strategic context. Injected automatically into every new session.

CLAUDE_MEMORY.md
~/SaSMaster/CLAUDE_MEMORY.md
The operational ground truth for Claude Code sessions. Loaded as the very first command in every terminal session — before any code is written. Contains architecture state, S3 paths, known issues, recent builds, and what's live vs planned.

SELF-LEARNING

The system doesn't just execute — it observes its own outputs, captures successful patterns as new skills, and improves with every build cycle. JARVIS's self-scoring queue closes the loop from task to learning.
