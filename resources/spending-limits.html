Active cost-guard.sh — Sourced in sasmaster-build.sh

Runs cost_guard_precheck() before every Claude invocation. Sums tokens_in + tokens_out from today's entries in logs/cost-log.jsonl. Returns exit 1 (abort) if daily total ≥ 250K.

After each build, cost_guard_write() appends the actual token + cost data to cost-log.jsonl and re-checks thresholds, posting Slack alerts only on threshold crossings (not every call above threshold).

Runbook Check Today's Token Usage 
# Sum today's tokens from cost-log.jsonl
python3 -c "
import json, datetime
today = datetime.date.today().isoformat()
total, entries = 0, 0
with open('$HOME/SaSMaster/logs/cost-log.jsonl') as f:
    for line in f:
        try:
            d = json.loads(line)
            if (d.get('timestamp','') or d.get('ts','')).startswith(today):
                total += int(d.get('tokens_in',0) or 0)
                total += int(d.get('tokens_out',0) or 0)
                entries += 1
        except: pass
print(f'Today: {total:,} tokens across {entries} entries')
print(f'Warn at: 150,000 | Abort at: 250,000')
"

Budget Philosophy
Target Aug 2026 Revenue-Before-Burn Crunch

SaSMaster must reach first B2B revenue by Aug 2026. Token spend controls are not about being cheap — they prevent runaway autonomous loop costs during overnight builds.

The 250K daily token abort ≈ ~$0.75–3.75/day depending on model mix. At current burn this is manageable. Revisit thresholds if autonomous loop cadence increases beyond 3 builds/day.
