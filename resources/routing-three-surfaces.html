5 token optimization rules
1Never run execution work in Claude.ai — you pay Sonnet prices for Haiku work
2Batch task cards in one session — 5 cards at once, not 5 separate sessions
3JARVIS queue for bulk = minimal context per task, cheaper per unit
4Haiku sub-agents for scraping, QA, data transforms — same output at 10× less cost
5Opus only for architecture decisions — most tasks don't need it
04 — Parallelism rules
PARALLELTasks share no files or state — e.g. RESEARCH-PORTAL-001 + LINKEDIN-001 can run simultaneously
PARALLELSub-agents in Claude Code operating on separate data sources — scraper-debugger + qa-agent
PARALLELSame token cost, half the wall clock — always worth it for independent tasks
SERIALTask B reads output of Task A — never parallelize dependent tasks
SERIALBoth tasks write to TASKS.md or index.html — last-write-wins = data corruption
SERIALDeploy gates — e.g. JARVIS-BATCH-001 must ship before RESEARCH-PORTAL-001 Ph2 starts
Parallel status — May 2026

True parallel execution (AGENT-PARALLEL-001) is not yet built. Current parallel = two independent Claude Code sessions running simultaneously on separate workstreams. No shared file writes. Once AGENT-PARALLEL-001 ships, JARVIS will orchestrate multiple sub-agents concurrently via the POST /jobs/parallel endpoint.
