SASMASTER ID Generation · Parent ID & CID

— minted once, at write time, never recomputed

SASMASTER PARENT ID · Title-Level

The immutable identity anchor for every movie and TV series. Minted deterministically at write time from the strongest anchor present:

# Movies
parent_key = COALESCE(
  imdb_tconst,           # preferred · 'tt1234567'
  'tmdb_mov_' + tmdb_id  # fallback · TMDB-only titles
)
# TV Series
parent_key = COALESCE(imdb_tconst, 'tmdb_tv_' + tmdb_id)
1Anchor resolution — IMDB / TMDB / EIDR checked at the gate; strongest anchor wins.
2Mint once — computed at WRITE TIME, never in a query; ak_id (spine row) assigned once, never changes.
3Bind alt IDs — EIDR, OPUS, GN rootId, Nielsen program IDs attach to the Parent ID via the alt-ID service (TALL format).
4Certify — anchor-based confidence score + band assigned; SCD2 insert to the spine. Never null, never truncated.

SASMASTER CID · Child / Content-Level

The child identity under each parent. A TV Series Parent ID is a shell — it carries no airing or version detail itself. CIDs hold the granular reality:

◆Episodes — every episode gets a CID nested under its series Parent ID, associated via series · season · episode mapping (IMDB episode file ↔ source schedules). Powers Episodic Nested metadata.
◆Versions & editions — language versions (e.g., Spanish primary-language / alt-title rows), cuts and ratings variants map to the same Parent ID with distinct CIDs — duplicates collapse instead of polluting the spine.
◆Airings — every matched telecast references Parent ID + CID, so viewership joins roll up cleanly: airing → episode → season → series.
# Identity hierarchy
SASMASTER_PID  tt0108778    # Friends · series shell
  └─ CID tt0583459  # S01E01 · episode
  └─ CID …          # versions · language · cuts
# Rule: viewership binds to CID · metadata rolls up to PID

Gracenote binding rule: parent ↔ rootId, never tmsId (version-specific).

The Matching Cascade

— alternate IDs first · data science second · humans last · higher tiers always win
TIER 1Asset Identification
Every inbound record is classified and anchored against the registries first — IMDB · TMDB · EIDR. Type resolution (movie · tv series · episode · special · live) plus deterministic identity where an anchor ID is already present. Paid programming filtered out.
→ typed + anchored at the gatemovies & tv = primary focus
▼ unresolved falls through ▼
TIER 2Schedule-Based Match
Linear airings married across Nielsen ↔ Gracenote ↔ FYI on same network + same air date + start/end time within ±8 min, timezone-normalized (Nielsen EST vs GN UTC). Future state: VideoAmp, Comscore, Kantar plug into the same join. The workhorse for viewership-to-metadata fusion.
✓ FULL = 1 · PARTIAL = 2time-window join · daily
▼ unresolved falls through ▼
TIER 3Alt-ID Lookup
Deterministic resolution through the alternate-ID service — SASMASTER_PID ↔ IMDB · TMDB · EIDR · OPUS · GN rootId · Nielsen program IDs. Known IDs resolve instantly with no recomputation and no guessing; newly observed IDs append to the mapping for next cycle.
✓ FULL MATCH → stagingzero-cost · O(lookup)
▼ unresolved falls through ▼
TIER 4API + Fuzzy Science
TMDB Search API (title + year — never primary-release-year) and EIDR API resolution, then composite fuzzy scoring: token_sort/set · Levenshtein ≤ 1–2 · year ±1 · actor(P=1) · prod-company · episode count. Runtime & director excluded — proven unreliable. Gracenote OnConnect resolver slotted, flag-gated.
✓ scored → confidence bandsrecovered = status 4
▼ only the hardest residue reaches humans ▼
TIER 5Human Adjudication
The interactive UI for human matching and auditing — partial / unmatched records surface in queues; admins search the spine, match, unmatch, approve. Every action is flagged manual_match, applied in the overnight cycle, and remembered forever in Master Asset Mapping.

Match Confidence Scoring

— strongest-anchor base + corroboration · certified is EIDR-gated

The Scoring Model · v2

Every fused record is scored by its strongest anchor, then corroborating anchors add on top. No amount of crowdsourced agreement can fake certification.

Base Score — set by strongest anchor present
EIDR presentcertified-only registry = ground truth0.95
IMDB present, no EIDRthe deterministic mint anchor0.50
TMDB only — orphanno IMDB/EIDR · weak; rare0.30
Corroboration — each additional anchor that agrees
TMDB agrees on an IMDB-base row+0.32
Gracenote / FYI agrees+0.15
Any anchor on an EIDR-base row+0.05
Anchor conflict — resolves to different titlesalso flags for manual review−0.30
Two structural rules
Certified is EIDR-gated. Non-EIDR rows are hard-capped at 0.89 — crowdsourced agreement can reach Confirmed but never Certified. Only the registry certifies.
imdb + tmdb clears Provisional comfortably → lands at 0.82, solidly Confirmed, not teetering on the 0.80 edge.

Audit Experience & Telemetry

— autonomous first · self-service always · communication is key

The Interactive Audit UI

◉
Match health by data stream — live match %, residue counts and trend per source (Nielsen, Gracenote, FYI, Avails…), refreshed every cycle from real telemetry — no static numbers.
▦
Drill-down grids — fully / partially / unmatched records, filterable by stream, network, date range, TV Series / Movies.
⚲
Manual match — search the spine (program + episode), select, confirm. Applied overnight with a manual_match provenance flag.
✂
Unmatch / unassociate — every decision reversible; the audit trail keeps both states.
⛨
Role-gated — all admins view streams; only platform admins hold read/write on the master spine.
⟳ The compounding flywheel: every adjudication writes back to Master Asset Mapping — tomorrow's residue checks yesterday's decisions first. The layer learns over time and asymptotically approaches 99% autonomous matching.

Telemetry & Slack Integration

📡
Run summaries — every matching cycle posts records processed, match rate by tier, new SASMASTER IDs minted, and residue delta to #builds via JARVIS.
⚠
Anomaly alerts — match-rate drops, source schema drift, volume spikes and conflict-flag surges page immediately. No silent failures, ever.
✋
Approval requests — bulk promotions (e.g., Provisional → Confirmed cohorts) and threshold changes require an explicit ✅ in Slack before applying.
🧠
Learning signals — weekly digest of what the engine learned: recurring manual-match patterns promoted to automated rules via the lessons-feedback pipeline.

Governance & Guarantees

— what makes it defensible under diligence
✓ Certified-Only Product
Only CERTIFIED (EIDR-gated) records reach shippable surfaces. Confirmed consumable with monitoring; Provisional held; Held adjudicated.
✗ Provenance Wall
No fabricated IDs, ever. Trial-observed Gracenote data lives in eval only. rootId binds to spine — never tmsId.
⛁ Immutable Spine
SCD2 append-only. SASMASTER PID minted at write time, never in a query. Full history preserved for every match decision.
⚖ Auditable by Design
Every match carries anchors, score, band, method, candidate count and timestamp. Telemetry per stream, per partner, per run.
