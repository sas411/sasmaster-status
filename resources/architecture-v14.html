Distributed compute
Live · 1.59× proven
Three-surface routing architecture
Parallel execution pattern
Domain-split only — never layer-split. Ad Intel ∥ Market Intel = valid. ui-designer ∥ data-viz = invalid (not independent). Serial baseline measured before any benchmark. 1.59× speedup proven May 25.
JARVIS write gate
JARVIS is the sole write gate to tasks.db (SQLite WAL). cost-log is agent-namespaced during parallel windows; JARVIS merges at join gate. JARVIS-BATCH-001: POST /tasks/batch on Railway commit 573cb5f.
TASKS.md gate syntax
Sequential tasks require explicit GATE: mark [x] done + verify before NEXT-TASK-ID and DEPENDS ON PREV-ID [x]. Prose dependency descriptions are not enough — JARVIS needs the [x] state check.
3-tier compute architecture
Tier 1 — Railway: Control plane, always-on. HTTP Events, status.json cron via /internal/generate-status, task routing + batch.
Tier 2 — Mac 64GB: Primary compute worker. Heartbeat-registered at /workers/heartbeat, polls /tasks/pending-compute, capabilities: duckdb / scraper / claude-code / ml.
Tier 3 — GitHub Actions: Scheduled fallback — fires when Mac worker offline or unreachable.

UI/UX architecture
LIVE
UI/UX architectural approach — from design token to rendered pixel
Component lifecycle
claude.ai artifact (design + concept) → /sandbox/ isolated file (build + QA + screenshot) → Storybook story (variants + baseline) → portal integration (wiring only, never redesign). No component skips the sandbox gate.
GPU compositor lanes
Lane 1: CSS compositor (UI micro-animations). Lane 2: Canvas 2D (chart rendering). Lane 3: WebGL (OGL animated backgrounds, ECharts-GL). Lane 4: Hardware video decode (HLS.js). M1 Max: zero CPU↔GPU transfer bottleneck — unified memory.
Design system tokens
Dark portal: --bg-base #0a0a0f, --lavender #a78bfa, --teal #22d3ee, --gold #f59e0b. Fonts: Bebas Neue / Barlow Condensed / Share Tech Mono. Style Dictionary: one JSON → CSS vars + TS types + Tailwind config.
