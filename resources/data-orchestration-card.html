End-to-End Data Orchestration

SaSMaster's data layer is a certified identity spine for all media content — IMDB, TMDB, Wikidata, EIDR, Nielsen, and Gracenote — unified via a single, immutable Parent Key. This walkthrough covers every stage from raw ingestion through certified fusion.

Spine Pattern
Master Pipeline — All Datasets
The core principle: Every dataset — IMDB, TMDB, Wikidata, EIDR, Nielsen, Gracenote — resolves to a single Parent Key before any JOIN. The key is computed once at write time, certified through a 5-signal model, and the spine is append-only (SCD2). Queries never recompute identity.

Delta Gate · Confirmed Changes Only

Before matching or certification, every dataset passes through a Delta Gate. Only genuinely new or changed rows proceed. This makes the pipeline O(delta) not O(full-catalog).

🎯 The EXCEPT Query

-- Confirmed delta: new + changed rows only SELECT * FROM raw_today EXCEPT SELECT * FROM parent_keys_spine -- Result: rows genuinely new or changed -- Unchanged rows → discarded -- Never re-certified if already stable

🔀 Per-Source Delta Mode

IMDB
Full dump → EXCEPT gateIMDB overwrites raw daily. EXCEPT extracts only the real delta.
TMDB
Changes API → 14-day windowAlready a changes-only feed. EXCEPT gate is a safety check.
Wiki
Resolve mode → held keys onlyAgent pointed at held_keys.txt — only unresolved tconsts.
Why this matters: Without the delta gate, every full IMDB dump (millions of rows) would trigger re-certification on already-stable records. The EXCEPT gate makes the pipeline O(delta) — certifying only what genuinely changed since the last spine write.

Matching & Certification

The identity resolution core. Every cross-system ID match is certified against a 5-signal model. Collected ≠ certified. Every fused ID gets a tier before it can touch the spine.

🔑 Parent Key Formula (v1)

# Movies parent_key = COALESCE( imdb_tconst, # preferred: 'tt1234567' — stable, canonical, industry standard 'tmdb_mov_' + str(tmdb_id) # fallback: TMDB-only titles (upcoming, international) ) # TV Series parent_key = COALESCE( imdb_tconst, 'tmdb_tv_' + str(tmdb_id) ) # Invariants: # - parent_key is NEVER NULL # - eidr_id is NULL in v1 (Phase 1b populates it) # - ak_id (spine row ID) assigned once, never changes # - Computed at WRITE TIME. Never in query.

🌐 Wikidata 5-Signal Certification Model

Every Wikidata-sourced ID match is run through 5 gate checks. Any blocking flag → QUARANTINE. All clear with corroboration → CERTIFIED.

SignalSourceWhat it checksVerdict
Rank wikibase:rank Is the claim still active or deprecated? QUARANTINE if deprecated
Entity Cardinality COUNT(P345) on QID Does the Wikidata entity carry >1 distinct IMDB ID? (merge candidate) QUARANTINE if >1
Cross-Entity Cardinality COUNT(DISTINCT ?item) per IMDb Is one IMDB ID claimed by >1 QID? QUARANTINE if >1
Type Agreement P31 (instance of) Is the entity actually a film/TV/episode? (Guards nm/ch IMDB trap — Q5=human) QUARANTINE if wrong type
Sourcing / Corroboration prov:wasDerivedFrom Is the claim referenced? Are ≥2 corroborating systems present on the QID? PROVISIONAL if unsourced
✅ CERTIFIED

All 5 signals pass. Claim not deprecated. Entity is 1:1. Type matches film/TV class. Sourced with ≥2 corroborating systems on the QID.

→ Auto-eligible for spine merge + eidr_id backfill + alt_ids promotion
~ PROVISIONAL

No blocking flags, but unsourced or sparse corroboration. Clean but unverified. Lands in fusion/ with fusion_tier='PROVISIONAL'.

→ Held. Sample-audited before any bulk promotion to spine.
✗ QUARANTINE

At least one blocking flag — conflict, merge candidate, wrong type, or deprecated claim. Lands in wikidata_prd/quarantine/.

→ Human adjudication queue ONLY. Never auto-promoted to spine.

📊 Parent Key v1 Schema

-- parent_keys/v1/YYYY-MM-DD/movies.parquet -- parent_keys/v1/YYYY-MM-DD/tv_series.parquet CREATE TABLE parent_keys_v1 ( parent_key VARCHAR NOT NULL, -- never null imdb_tconst VARCHAR, -- NULL if TMDB-only tmdb_id INTEGER, tmdb_type VARCHAR, -- 'movie' | 'tv' eidr_id VARCHAR, -- NULL in v1 → Phase 1b title VARCHAR, year INTEGER, ingested_at TIMESTAMP ); -- ZSTD level 3 · row group 128MB target -- INSERT ONLY. Never UPDATE or TRUNCATE. -- MAX(ingested_at) = current state per key

🔄 SCD2 Write Rules

✓
Insert new rows for changed recordsPreserves full history. Use MAX(ingested_at) for current state.
✗
Never overwrite existing partitionsWrite to new YYYY-MM-DD. Old partitions are immutable.
✗
Never truncate the spineEven for full rebuilds. Append-only is the contract.
✓
Movies and TV always separate filesDifferent join semantics. Never merge into one Parquet.
✗
No parallel spine writesSerialize: Wikidata resolve + IMDB resolve cannot run simultaneously.

📐 Schema Evolution Protocol

✅ Allowed without version bump
Add new nullable columnNULL default, backward compatible
Widen numeric typeint32 → int64. Safe.
New YYYY-MM-DD partitionAppend-only by design
🚫 Requires v1 → v2 bump
Remove or rename columnBreaking change for all readers
Change column typeCast in query layer, never in storage

Version bump protocol: Write new path (parent_keys/v2/…) → run v1 and v2 in parallel for ≥30 days → migrate all readers (portal, war room, analytics) → flip v1 to Glacier.

Fusion & Marts Refresh

After the spine write, cross-dataset JOINs are computed via parent_key. DuckDB is the sole compute engine — no Spark, no Pandas for large sets. All joins happen on the certified key.

⚙️ Canonical Cross-Dataset JOIN

-- Always join via parent_key. Never join on raw IDs across datasets. SELECT pk.parent_key, pk.title, pk.year, pk.eidr_id, -- NULL in v1, populated v1b+ tmdb.popularity, tmdb.vote_average, wiki.netflix_id, wiki.rottentomatoes_id FROM read_parquet('s3://sasmaster-2026/parent_keys/v1/2026-06-09/movies.parquet') pk LEFT JOIN read_parquet('s3://sasmaster-2026/tmdb_dev/2026-06-09/movies.parquet') tmdb ON pk.tmdb_id = tmdb.id LEFT JOIN ( SELECT parent_key, MAX_BY(id_value, ingested_at) FILTER(WHERE id_system = 'netflix_id') AS netflix_id, MAX_BY(id_value, ingested_at) FILTER(WHERE id_system = 'rottentomatoes_id') AS rottentomatoes_id FROM read_parquet('s3://sasmaster-2026/wikidata_prd/alt_ids/2026-06-09/data.parquet') WHERE anchor_tier = 'CERTIFIED' -- only certified alt-IDs reach product GROUP BY parent_key ) wiki ON pk.parent_key = wiki.parent_key;
