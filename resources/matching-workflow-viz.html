OPUS Resolution Paths — 4 Decision Branches

1
Title Disambiguation
Input: multiple tconst candidates for normalized title+year. OPUS scores each on cast overlap, runtime delta, release geography, and source authority weight. Output: single winning tconst or QUARANTINE if score below threshold.
→ BIND_TCONST→ QUARANTINE
2
Conflict Adjudication (P3)
Input: 138 records where EIDR edition data conflicts with existing tconst binding. OPUS reviews EIDR vs IMDB authority chain. Output: BLESS (keep existing), REPLACE (swap tconst), or ESCALATE (human review flag).
→ BLESS→ REPLACE→ ESCALATE
3
Candidate Promotion
Input: 5,142 EIDR-only partial records. When EIDR auth activates (target: 70–75% coverage), OPUS re-evaluates each for promotion from PARTIAL → CERTIFIED. Checks uniqueness cert (1:1 gate) before promoting.
→ PROMOTE_TO_CERTIFIED→ HOLD_PARTIAL
4
Blind Spot Detection
Post-ingest sweep: Haiku scans for titles that passed Phase 1+2 with no match. Outputs structured candidate list with suggested search strings. Routes high-confidence candidates to Sonnet for verification, escalates remainder to OPUS.
→ AUTO_MATCH→ VERIFY_QUEUE→ UNRESOLVED

GenAI Output Types

BIND_TCONST
Confirmed tt####### binding written to parent_keys/v2. Immutable after cert gate.
QUARANTINE
Record isolated in quarantine/. 17 wrong-tconst + 121 EIDR-edition pending. Not in v2.
ESCALATE
Human review flag posted to Slack #eidr-conflicts with full candidate context.
PROMOTE
PARTIAL → CERTIFIED when uniqueness cert passes and EIDR auth gate is open.
UNRESOLVED
Record written to unresolved/staging/ with full candidate list for future resolution.
BLESS
Existing tconst binding confirmed correct after OPUS adjudication. No write needed.
