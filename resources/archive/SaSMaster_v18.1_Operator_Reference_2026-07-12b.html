The ASK Plan Grammar — How Operators Query the Platform P2 · 30/30 GREEN

ASK is not "an LLM with database access" — it is a constrained plan grammar: the model may only compose plans the grammar manifest permits, the validator rejects anything else before execution, and the executor applies the band + licensing predicates as a physically unbypassable last gate. The grammar IS the product boundary — what the platform will answer is defined in data, not in prompt engineering.

ASK platform P2 shipped · executor P4 on Railway · ask_ro share refreshed 07-12 · grammar manifest = the product boundary, versioned as data

📐 Why a grammar, not a prompt

Prompts drift; grammars don't. Every answerable question maps to a plan the manifest already permits, so the answer space is enumerable, testable, and certifiable — the same property that lets the Reporting Catalog trace every figure to a certification row.

THE PRODUCT BOUNDARY

🛡 The unbypassable gate

The CERTIFIED+CONFIRMED band predicate and licensing_class envelope are injected at the executor — below the model, below the plan. Even a hypothetically jailbroken planner cannot select an uncertified or unlicensed row.

BLUEPRINT §8 · EXIT CRITERION

🔁 Grammar meets flywheel

Blessed answers deposit as gold examples against their plan; patterns seen ≥3 times promote to catalog entries. The grammar grows by governed accretion — every expansion is a reviewed manifest change with a test, never a silent capability.

Lessons Learned — The Complete Ledger 001–017

v17's lessons (001–005, condensed) established the sourcing doctrine. The certification era added eight, and the v18.1 revision session itself added three more (015–017) — each paid for with real incident time, each now enforced by structure rather than memory.

ARCH-LESSONS 001–005 — the v17 sourcing doctrine (in force)

001 Never estimate what you can query — live query, War Room, or confirmed session; never memory. 002 Parent Key counts have four meanings (total / PKIDs / CIDs / source rows) — always state which. 003 Fleet counts come from their source of truth, never the claude.ai mount. 004 Every data point carries a source badge; unsourced = "pending verification," never a guess. 005 The Bloomberg standard: every public artifact session begins with a live data pull.

ARCH-LESSON-006 — A re-mint without a downstream sweep is a time bomb

The 06-20 spine re-mint shipped with zero downstream verification: a crosswalk sat at 0.0% match for 19 days, a seed batch landed at 25.9%, and the ASK share served pre-remint data until it broke in production. One systematic sweep found all three in an afternoon.

Every spine re-mint closes with a match-rate check on every PKID-joining table (§46d); every write job carries a remint-boundary guard. Identity changes propagate by verification, never by assumption.

ARCH-LESSON-007 — Placeholder strings are not match evidence

primaryTitle='N/A' matched 660,276 telecasts — 83% of the viewership bridge. The bridge looked rich; it was mostly noise wearing a match's clothing.

Junk-title classes live in a data-driven blocklist the matcher must consult; suppressions counted visibly per run; rows-per-pkid skew trips review before write. A smaller honest number always beats a larger fabricated one.

ARCH-LESSON-008 — Materialized copies rot; views break loudly

The ASK share froze at its 06-09 snapshot because its update mode is MANUAL and nobody knew — serving 903K movies when reality was 2.32M, on 9 columns when the table had 17.

Serve via views over current-mint objects wherever mechanics allow. Where a copy is forced: UPDATE SHARE after every data release, freshness check in the weekly cert run, rebuild trigger on any re-mint.

ARCH-LESSON-009 — State the grain or the number lies

The first cert pass reported 101,585 "dups" on a clean spine — SCD2 version history counted at all-rows grain. Separately, org-type EIDR domains showed thousands of "dups" that are legitimate at their correct grain.

Every count states its GROUP BY key (§46b); per-domain grains live in a registry (ops.altid_domain_grain) — declared as data, never hardcoded assumptions.

ARCH-LESSON-010 — Dimensions are columns, not table families

Nine per-stream viewership shells were scaffolded on the 2018 pattern. In a columnar engine, table-per-stream buys zero pruning and costs nine-times-everything — and the forgotten ninth copy is the next rot instance.

Low-cardinality dimensions (viewing_stream, source_system) are partition columns in one table per view type. New streams are a row in ref.ref_viewing_stream, never a new table. Ruled at zero rows — the cheapest decision ever made.

ARCH-LESSON-011 — Match engines to their economics

MotherDuck hit its daily compute cap on certification queries alone. The fix wasn't a $250/month plan — it was routing: atomic scans local and free, serving small, and a dual-emitted embedded warehouse.

MotherDuck never scans atomic-scale data. The serving warehouse dual-emits (free tier + embedded file, byte-equivalence certified). Paid tiers require proof of real concurrent load — not a throttle warning.

ARCH-LESSON-012 — Test the tripwire, then test the test

The reconciliation tripwire's first forced-mismatch attempt corrupted a key that didn't exist — matched zero rows — and the cert correctly stayed PASS. An untested test would have been declared proven while proving nothing.

Every guard ships with a forced-fire test, and the test verifies its own preconditions. "Proven" means the failure was demonstrated, not that the check ran green.

ARCH-LESSON-013 — Attached shares contaminate catalog queries

A schema inspection showed phantom duplicate columns on a clean prod table — the attached ask_ro share was echoing every information_schema row. An hour of anomaly-chasing, zero defects.

Catalog and certification queries filter to database_name explicitly (or detach shares in the runner session). Share-echo rows can never contaminate a certification result.

ARCH-LESSON-014 — Load the brand system before building any artifact

The first two v18 drafts shipped in an invented theme — dark, then an off-palette light — because the session lacked the design skills and defaulted from generic memory instead of the canonical system. The five locked themes and the BRAND-003 light-default ruling existed the whole time in Brand Reference v8.1 and past sessions.

Every artifact session begins by loading the brand system (Brand Reference v8.1 / design-skill tokens) exactly as data sessions begin with a live query — ARCH-LESSON-005's twin for design. Theme 02 LIGHT is the default; Theme 01 DARK is scoped to Portal/War Room. No artifact ships on reconstructed-from-memory tokens.

ARCH-LESSON-015 — Reconstruction is not restoration

A "rerun" request was answered by regenerating v18 from session fragments and memory instead of retrieving the deployed file. The rebuild looked complete but silently dropped four whole sections (KB Architecture, Spotlight, Catalog, Card Queue), lesson 014, and the canonical font system — LESSON-014 repeated by a different route.

When a canonical artifact exists (deployed, on disk, in the vault), retrieve it before regenerating anything. A rebuild from fragments is a draft, never a replacement, and must be labeled as such.

ARCH-LESSON-016 — A source badge without a grain is still a lie

A live, badged MotherDuck query returned 13,090,272 spine rows and was presented as the count; the correct figure was 12,955,801 — the query was raw SCD2 all-rows, superseded versions included. The number was live, sourced, and wrong: LESSON-009's twin with a new attack surface, this time from the assistant rather than the pipeline.

The badge must carry the grain (e.g. "MOTHERDUCK · is_current · 07-12"). Any spine query without an explicit is_current stance is invalid by default — for humans, agents, and assistants alike.

ARCH-LESSON-017 — Visual artifacts need a certification gate too

The shipped v18 had SVG text overflowing its boxes and a v17 capability silently downgraded (the animated Spotlight became static swatches); neither was caught before an operator screenshot. A programmatic text-width audit against each viewBox then caught an additional overflow nobody had flagged.

Every visual artifact ships through (a) an automated text-fit audit per viewBox, (b) a version-over-version feature parity diff, and (c) screenshot review at desktop and mobile widths. The qa-agent gate applies to documents, not just portal code. Rotated SVG labels are middle-anchored so font fallback can never clip them.
