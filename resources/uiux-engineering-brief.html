New Component Protocol — Mandatory 7-Gate Checklist
1Build in Sandbox First

Create sandbox/<component>-sandbox.html. Must render standalone at file:// — no server, no imports, tokens inline.

2QA Agent Pass

Run node scripts/qa-agent.js sandbox/<component>.html. Fix all failures before proceeding.

3Write the Story First

Add packages/design-system/stories/<Component>.stories.js with Default + Loading + Error variants minimum.

4Storybook Renders

Run npm run storybook in packages/design-system/. All variants must render on dark background at port 6006.

5Write CONTEXT.md

Create <Component>.CONTEXT.md alongside the story. Document tokens used, hard constraints, edge cases. Template at docs/CONTEXT_TEMPLATE.md.

6Zero Token Violations

Run bash scripts/audit.sh. Gate must exit 0. Use token-exempt-block-start/end for brand/screen accent colors only.

7Only Then: Copy to Portal

All 6 gates above must pass before component logic is integrated into portal. No exceptions. Write a DDR for any decision that would surprise a future developer — template at docs/DDR_TEMPLATE.md.

Design Decision Records — docs/decisions/
DDR-001
D3 Force Simulation Retirement
Removed force-directed graphs — 40% CPU reduction, physics adds no analytical value for media intelligence use case. Do not revert: performance budget is strict; any graph need can be met with Sankey/amCharts without physics sim.
DDR-002
Dr. Scoop Component Isolation Pattern
Dr. Scoop fetches its own data, manages its own state, accepts only a userId init call. Prior wiring caused cascading re-renders. Isolation is what makes it embeddable across portal, War Room, and research portal without refactor.
DDR-003
Dark Cinematic Token System — Canonical Adoption
All portal UI uses exclusively tokens.md v1.0. Hardcoded hex/rgb banned from active source — enforced by audit.sh build gate. Prior state: 300+ unique color values, no shared vocabulary, style changes required multi-file find/replace.
