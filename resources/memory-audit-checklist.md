# Memory Audit Checklist — SaSMaster

**Sprint:** HARDEN-009  
**Date:** 2026-05-24  
**Owner:** Shiv Sehgal

## Purpose
Periodic security + hygiene audit across all three memory layers:  
1. Claude Code session memory (`~/.claude/projects/…/memory/`)  
2. DuckDB persistent memory (`scoop_memory` table, `sasmaster.duckdb`)  
3. S3 memory backup (`s3://sasmaster-2026/scoop/memory/`)

Run this checklist quarterly or after any credential rotation / major sprint.

---

## Layer 1 — Claude Code Session Memory

### Credential Safety
- [ ] Grep all memory files for patterns that look like secrets:
  ```bash
  grep -rEi 'key|token|secret|password|bearer|sk-|xoxb|xoxp' \
    ~/.claude/projects/-Users-shivashish/memory/ \
    --include="*.md" -l
  ```
  **Pass:** zero files returned, or returned files contain only key *names* (e.g., `AWS_ACCESS_KEY_ID`) not values.

- [ ] Verify MEMORY.md index is under 200 lines (truncation risk):
  ```bash
  wc -l ~/.claude/projects/-Users-shivashish/memory/MEMORY.md
  ```
  **Pass:** < 200 lines.

### Content Hygiene
- [ ] Stale project memories — any memory referencing tasks/dates > 90 days old should be reviewed for archival.
- [ ] No duplicate slugs in MEMORY.md (grep for repeated `- [` entries with same file path).
- [ ] No memory files reference RSG Media, Rightsline, or prior employer systems.

## Layer 2 — DuckDB scoop_memory Table

### Schema Integrity
- [ ] Table exists and schema is current:
  ```bash
  duckdb ~/SaSMaster/data/sasmaster.duckdb -c ".schema scoop_memory"
  ```
  Expected columns: `user_id, last_session, session_count, company_focus, metric_focus, last_insights_surfaced, last_briefing_date, preferred_depth, open_questions, updated_at`

- [ ] Row count is reasonable (< 1000 for a personal system):
  ```bash
  duckdb ~/SaSMaster/data/sasmaster.duckdb -c "SELECT COUNT(*) FROM scoop_memory;"
  ```

- [ ] No NULL user_ids:
  ```bash
  duckdb ~/SaSMaster/data/sasmaster.duckdb -c "SELECT COUNT(*) FROM scoop_memory WHERE user_id IS NULL;"
  ```
  **Pass:** 0.

### Data Safety
- [ ] `open_questions` and `last_insights_surfaced` contain no raw credentials (spot check 3 rows):
  ```bash
  duckdb ~/SaSMaster/data/sasmaster.duckdb -c \
    "SELECT user_id, open_questions FROM scoop_memory LIMIT 3;"
  ```

- [ ] Database file is not world-readable:
  ```bash
  ls -la ~/SaSMaster/data/sasmaster.duckdb | awk '{print $1}'
  ```
  **Pass:** permissions are `-rw-------` or `-rw-r-----` (not `o+r`).

### Connection Health
- [ ] QueryAdapter closes connections on shutdown (check railway.ts for `close()` call in shutdown handler).
- [ ] No stale lock files:
  ```bash
  ls /tmp/*.lock 2>/dev/null | grep -E "scoop|duckdb|sasmaster"
  ```

## Layer 3 — S3 Memory Backup

### Freshness
- [ ] Most recent backup is within the last 7 days:
  ```bash
  aws s3 ls s3://sasmaster-2026/scoop/memory/ --recursive \
    | sort | tail -3
  ```

- [ ] Backup files are non-zero:
  ```bash
  aws s3 ls s3://sasmaster-2026/scoop/memory/ --recursive \
    | awk '{print $3}' | sort -n | head -3
  ```
  **Pass:** all sizes > 0.

### Access Control
- [ ] S3 bucket has public access blocked (verified in HARDEN-003 — re-confirm):
  ```bash
  aws s3api get-public-access-block --bucket sasmaster-2026 \
    | python3 -m json.tool
  ```
  **Pass:** all four `BlockPublicAcls/IgnorePublicAcls/BlockPublicPolicy/RestrictPublicBuckets` = `true`.

- [ ] No pre-signed URLs or bucket policies exposing scoop/memory/ path publicly.

## Layer 4 — S3 Entity Counts Cache

### Freshness
- [ ] `cache/api/data-counts.json` age < 30 days (generated by s3-entity-counts.js):
  ```bash
  aws s3 ls s3://sasmaster-2026/cache/api/data-counts.json
  ```
  Check the date column — if > 30 days, re-run `node ~/SaSMaster/scripts/s3-entity-counts.js`.

- [ ] `sync-to-s3-cache.sh` pushes data-counts.json on every 5-min tick:
  ```bash
  grep "data-counts" ~/SaSMaster/scripts/sync-to-s3-cache.sh
  ```

## Remediation Procedures

| Finding | Action |
|---------|--------|
| Secret value in memory file | Immediately `rm` the file, rotate the credential, update MEMORY.md index |
| MEMORY.md > 200 lines | Archive oldest project_*.md files to `~/SaSMaster/obsidian/` cold storage |
| scoop_memory schema mismatch | Run `initMemoryTable()` via JARVIS `!scoop diagnose` command |
| S3 backup stale > 7 days | Re-enable `backupMemoryToS3()` call in cron or run manually |
| data-counts.json > 30 days | `node ~/SaSMaster/scripts/s3-entity-counts.js` (takes ~12 min) |
| DB file world-readable | `chmod 600 ~/SaSMaster/data/sasmaster.duckdb` |
