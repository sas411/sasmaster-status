Lessons Learned — Architecture Document Integrity

This section exists because v17's first two drafts contained materially wrong data despite live sources being available. That is a failure of intelligence for a platform that calls itself a Bloomberg Terminal. These lessons are permanent.

ARCH-LESSON-001 — Never estimate what you can query
v17 drafts 1 and 2 showed "125.6 GB" and then "~200–400 GB est." for the S3 lake size. The War Room shows 1.62 TB live. MotherDuck was available the entire time. The failure was not querying before writing. Rule: Any number that appears in an architecture document must come from a live query, a War Room screenshot, or a confirmed session result — never from memory or estimation when live access exists.
ARCH-LESSON-002 — Parent Key counts have two different meanings; always state which
The document alternated between "861,878 parent keys" (the v1 IMDB spine count from a May evaluation session) and "12.9M rows" (the v2.0 full spine) without distinguishing them. Both numbers appear in session history. Rule: Always specify: spine total rows / PKIDs only (no CID) / episode CIDs / source dataset row counts. These are four different numbers. Query prod.parent_key with explicit WHERE clauses before writing. Current confirmed: 12,955,795 total · 3,286,241 PKIDs · 9,669,554 CIDs.
ARCH-LESSON-003 — Agent and skill counts have a persistent source-of-truth; always check it
v17 draft showed "50+ skills" and an invented fleet breakdown of "15 native / 31 marketplace." Ground truth: 79 skills on disk (architecture-v14.html, Claude Code confirmed Jun 5), 51 agents total (War Room live header). The claude.ai mount shows ~22 skills — this is a known mount artifact. Rule: Fleet counts always come from architecture-v14.html or the War Room AGENTS tab header. Never infer from what's visible in the claude.ai skills panel.
ARCH-LESSON-004 — Architecture artifacts must cite their data sources inline
The first drafts presented numbers with no provenance. A reader could not tell if "861,878" came from a live query or a hallucination. Rule: Every data point in an architecture document carries a source badge: MOTHERDUCK · TABLE, WAR ROOM · TAB, or SESSION · DATE. If a number cannot be sourced, it is replaced with "pending verification" — never a guess. This document implements this pattern.
ARCH-LESSON-005 — The standard for a "Bloomberg Terminal for Media" is higher than average
The platform's core value proposition is decision-grade data. An architecture document that presents wrong figures undermines that claim before a single line of product code ships. Rule: Every session that produces a public-facing artifact (architecture doc, War Room update, investor deck) must begin with a live data pull. The MCP connections (MotherDuck, War Room screenshots, Databricks) exist precisely for this. Using them is not optional — it is the intelligence standard.
