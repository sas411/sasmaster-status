02 — Five Contracts · Five Powers

Every capability plugs into exactly five contracts
  1. Serving API — one authenticated HTTP surface, fail-closed
  2. S3 task bus — queue, heartbeat, catch-up; idempotency keys
  3. Model gateway — sole non-Claude egress, write-ahead ledger
  4. Routing gate — owner skills + doctrine resolved before any edit
  5. Cert gate — builder ≠ verifier ≠ certifier; weekly certs, git-SHA pinned
The control plane, as five separated powers
  • Law — CLAUDE.md: thin-pointer constitution; invariants + startup only
  • Knowledge — the Brain: L2 context + L3 state; facts pulled, never carried
  • Will — JARVIS kernel: the executing agent OS (shadow; graduates on measured accuracy)
  • Conscience — the gate mesh: physically blocks ungoverned writes
  • Sovereign — Shiv: the only push authority; weekly bless window + gated fast lane
