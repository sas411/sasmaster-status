02 — The Four Interop Rails — what makes the cartridges interchangeable

S3 TASK BUS
every job, queue, heartbeat and shadow-diff speaks S3 — hosts are interchangeable because the bus, not the host, owns the work.
L3 STATE
one MotherDuck platform_state row per fact — any surface, agent or session rehydrates from the same truth. Counts pulled, never carried.
EGRESS LEDGER
every external-model call writes its allow/block row atomically — new models inherit the clean-room wall by construction.
LEARNING RAIL ▲
Dreams → vault → skills → hooks. Every module's mistakes become every future module's doctrine — the fabric compounds nightly.

03 — Why This Shape

Plug-and-play is a property of contracts, not components. The audit showed modules already churn (models, vendors, hosts, surfaces) — what breaks is everything wired around them. The future state hardens the five contracts (serving · task bus · gateway · routing gate · cert gate) so churn becomes cheap: a new Nielsen competitor is an adapter, a new frontier model is a gateway row, a second worker is a bus consumer, a B2B portal is a serving client. Nothing above or below a socket knows what cartridge is in it — that is the whole design.
